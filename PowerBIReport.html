PowerBI Report on OECD Well-Being Data

This report for a data science module in university showcases different economic and social indicators from all OECD countries. The idea behind this project was to find a suitable data set and then create a interactive Power BI report to allow others to explore the data set.

My group and I settled on the OECD data set as it was high quality data and allows for different visualisations. We initially checked the data set for outliers, but all outlier values where explainable and externally confirmed. The data set contains 31 indicators we grouped into economic, employment, education, pollution, health, safety, geography and others. For the PowerBI report the topics where again grouped together to cover them on six pages. On the left we included a navigation bar and on the top right we included fiter panels.

Dashboard #1: Overview
This dashboard provides a overview on OECD countries GDP, GDP/CAP and population. We chose a map visualisation coupled with a set of key figures to represent the economic overview of OECD countries. The map provides a geographical context, while the use of differently sized circles enables a quick visual comparison between countries. The key figures above the map – Total OECD GDP, Weighted Average OECD GDP/Capita, and Total OECD Population – offer high-level summary statistics that provide a snapshot of the economic scale and individual prosperity within the OECD. While the map changes depending on the filter selection, these panels always display the same.

Dashboard #2: Economics
The second page uses summary statistics and a bar chart to display the OECD countries economic and living situation. On the top you can find eight different filter options to look at different indicators. The summary statistics sets individual countries in the bar chart into context. Similarly the bar chart allows to directly compare OECD countries to each other. This is useful to quickly determine how countries rank in a certain measures.

Dashboard #3: Employment & Education
This page covers employment an education. We chose to divide the page into a bar chart and summary statistics for employment measures and education measures. You can choose the indicators you want to see and with this divided layout you can also compare your selected employment and education measures. On the top right you can choose what countries you want to see displayed on this page. You can filter by G7, BRICS, developed and developing countries and the regions like northern or central Europe.

Dashboard #4: Quality of Life
This page covers the quality of life through different safety, health and environment indicators. The page is filtered by developed and developing countries. The first visual on safety displays homicide and safety indicators. On the bottom health and environment are displayed over six different indicators. The way these indicators are displayed allows users to quickly see how countries rank in the different indicators. Together with the “Best” and “Worst” filter, this can be used to determine which OECD countries are the best or worst options in terms of quality of life.

Dashboard #5: Correlations between Indicators and their Implications
On this last page we used python code embedded in PowerBI to search for interesting relations in the data set, regarding personal earnings and life expectancy. We displayed the correlation between the measures in an interactive way. If you want to see a chart in more detail you can click on it and it will direct you to the full sized chart for easier readability.

Dashboard #6: Measures Wiki
This last dashboard explains some of the indicators that are not self-explanatory.

Overall this PowerBI report showcases my skills in creating interactive and well structured reports, which allow users to conduct their own data exploration. The process of the data selection was integral in creating a good report. When designing the report it was important to think about which visualisations allow for the easiest and most straight forward displaying of the data. Adding features like the navigation panel or the measures wiki where important to increase the usability.
